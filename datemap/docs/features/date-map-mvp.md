# Date Map MVP

## 목표

첫 번째 구현에서는 사용자가 장소를 검색해 위치를 선택하고, 같은 날짜 안에 여러 데이트 장소를 저장하며, 장소별로 여러 장의 사진과 한 줄 일기, 해시태그를 남길 수 있게 합니다.

## 사용자 흐름

1. 사용자가 앱을 연다.
2. 메인 화면에서 대한민국 지도 위의 하트 마커와 하단 저장 카드를 본다.
3. 하단 카드 목록은 자동으로 천천히 좌측 스와이프되며 최근 기록을 보여준다.
4. 사용자가 날짜 또는 기간을 선택하면 해당 기간의 하트 마커와 카드만 보인다.
5. 사용자가 장소 추가하기 버튼을 누른다.
6. 장소 기록 작성 팝업이 열린다.
7. 검색창에서 가게 이름이나 장소 이름을 검색한다.
8. 검색 결과에서 장소의 이름, 주소, 좌표를 가져온다.
9. 데이트 날짜를 캘린더에서 선택한다.
10. 같은 날짜 안에 장소를 하나 이상 추가한다.
11. 각 장소에 사진 여러 장을 업로드하고 대표 사진을 지정한다.
12. 각 장소에 한 줄 일기와 해시태그를 작성한다.
13. 저장 후 팝업이 닫히고 메인 화면으로 돌아간다.
14. 저장된 장소가 하트 마커와 하단 카드 목록에 나타난다.

## 핵심 화면

### 지도 홈

- 네이버 지도 기반 대한민국 지도 표시
- 저장된 장소의 위도/경도를 기준으로 방문 장소를 하트 마커로 표시
- 드롭다운 기반 날짜 프리셋 필터
- 캘린더 아이콘을 통한 모달형 DatePicker
- DatePicker에서 기간 선택 또는 주차 선택
- 저장 기록이 있는 날짜는 캘린더에서 활성 표시
- 하단 최근 데이트 카드 자동 스와이프
- 장소 추가 버튼

### 장소 추가 팝업

- 장소 추가하기 버튼으로 열린다.
- 날짜 캘린더
- 네이버 지도 위치 선택
- 장소 검색 또는 장소명 직접 입력
- 사진 여러 장 업로드
- 선택한 사진은 앱 내부 문서 폴더로 복사 후 로컬 URI 저장
- 대표 사진 선택
- 한 줄 일기
- 해시태그
- 저장 후 메인 화면으로 복귀

### 장소 선택

- 장소명 검색
- 검색 결과 목록
- 지도에서 직접 핀 지정은 보조 수단
- 선택한 장소 정보 확인

### 데이트 카드 작성

- 데이트 날짜
- 같은 날짜의 장소 목록
- 장소 추가 버튼

### 장소 기록 작성

- 장소명
- 주소
- 사진 여러 장
- 한 줄 일기
- 해시태그
- 저장 버튼

### 날짜별 보기

- 날짜 선택
- 해당 날짜의 장소 목록
- 같은 날짜에 방문한 여러 장소 표시
- 장소 선택 시 사진과 한 줄 일기 표시

### 지도 보기

- 대한민국 전체 지도
- 방문했던 장소를 하트 마커로 표시
- 주별 필터
- 월별 필터
- 장소 또는 키워드 검색 결과만 표시

### 저장 목록

- 중앙에 대한민국 지도 요약
- 네이버 지도 위에 방문한 장소를 하트 마커로 표시
- 하단에 저장된 장소 카드 목록 표시
- 카드에는 장소 이름, 방문 날짜, 대표 사진, 해시태그 표시
- 카드 목록은 천천히 좌측으로 스와이프되며 기록을 보여줌
- 정확한 날짜 또는 기간 선택 시 해당 기간의 카드만 표시
- 드롭다운으로 전체/이번 주/이번 달/최근 30일을 빠르게 선택
- 캘린더 모달에서 기간 또는 주차를 직접 선택
- 카드를 선택하면 상세 화면을 표시
- 상세 화면 우상단에 수정/삭제 액션 고정

## MVP 데이터

MVP는 날짜와 장소를 분리해서 저장합니다. 같은 날짜에 여러 장소를 저장할 수 있고, 같은 장소에는 여러 사진을 연결할 수 있어야 합니다.

데이트 일자 하나는 최소한 아래 정보를 가져야 합니다.

- 고유 ID
- 데이트 날짜
- 생성일
- 수정일

데이트 장소 하나는 최소한 아래 정보를 가져야 합니다.

- 고유 ID
- 데이트 일자 ID
- 장소명
- 주소
- 위도
- 경도
- 대표 사진 URI
- 한 줄 일기
- 해시태그 목록
- 생성일
- 수정일

사진 하나는 최소한 아래 정보를 가져야 합니다.

- 고유 ID
- 데이트 장소 ID
- 로컬 사진 URI
- 표시 순서
- 생성일

## 기능 요구사항

- 사용자는 장소를 검색해서 데이트 장소로 저장할 수 있다.
- 사용자는 지도에서 직접 위치를 지정해서 저장할 수 있다. 단, MVP의 기본 흐름은 장소명 검색이다.
- 사용자는 같은 날짜에 여러 장소를 저장할 수 있다.
- 사용자는 같은 장소에 사진 여러 장을 첨부할 수 있다.
- 사용자는 장소별로 한 줄 일기를 작성할 수 있다.
- 사용자는 장소별로 해시태그를 작성할 수 있다.
- 사용자는 한 줄 일기나 사진이 없어도 장소를 임시 저장할 수 있다.
- 사용자는 저장된 데이트 장소를 지도 마커로 볼 수 있다.
- 사용자는 저장된 데이트 장소를 날짜별로 볼 수 있다.
- 사용자는 추후 주별, 월별 필터로 방문 장소를 볼 수 있다.

## 현재 구현 상태

- 메인 화면은 네이버 지도와 저장 장소 하트 마커 중심으로 구성되어 있다.
- 하트 마커는 저장된 장소의 위도/경도를 기준으로 표시한다.
- 메인 날짜 필터는 드롭다운 프리셋과 캘린더 아이콘 모달로 구성되어 있다.
- 캘린더 모달에서는 기간 선택과 주차 선택을 지원한다.
- 저장 기록이 있는 날짜는 캘린더에서 활성 점으로 표시한다.
- 저장된 장소 카드는 하단에서 가로 목록으로 보이고 자동으로 좌측 스와이프된다.
- 장소 추가하기 버튼을 누르면 장소 기록 작성 팝업이 열린다.
- 작성 팝업에서 캘린더 날짜 선택, 지도 좌표 선택, 장소명 입력, 사진 여러 장 선택, 대표 사진 선택, 한 줄 일기, 해시태그 입력을 할 수 있다.
- 선택한 사진은 앱 내부 문서 폴더로 복사한 뒤 SQLite에는 복사된 로컬 URI를 저장한다.
- 저장 후 팝업이 닫히고 메인 화면의 장소 개수, 하트 마커, 카드 목록이 갱신된다.
- 카드 선택 시 상세 팝업이 열린다.
- 상세 팝업 우상단에 닫기, 수정, 삭제 액션을 고정 배치한다.
- 삭제는 SQLite의 장소와 사진 연결 데이터를 삭제한다.

다음 구현은 네이버 장소 검색 API 연결과 저장된 기록 수정 폼입니다.

- 장소 추가 팝업 안의 지도 화면
- 장소명 입력 필드
- 사진 여러 장 선택
- 사진 보관함 권한 요청
- 앱 내부 문서 폴더로 사진 복사
- 대표 사진 선택
- 한 줄 일기 입력
- 해시태그 입력과 칩 표시
- SQLite 저장 버튼
- 저장 후 메인 화면 복귀

추후 구현:

- 네이버 장소 검색 API 연결
- 저장된 장소 기록 수정 폼
- 사진 추가/삭제/대표 사진 변경

## 저장 목록 뼈대

현재 구현:

- 저장된 장소 조회
- 네이버 지도 기반 요약 패널
- 방문 장소 위치를 위도/경도 기준 하트 마커로 표시
- 드롭다운 프리셋 날짜 필터
- 모달형 DatePicker 시작일/종료일 기간 필터
- 모달형 DatePicker 주차 필터
- 저장 기록이 있는 날짜 활성 표시
- 저장된 장소 카드 가로 목록
- 카드 자동 좌측 스와이프
- 카드에 장소명, 날짜, 대표 사진, 해시태그 표시

- 상세 팝업 우상단 수정/삭제 액션 배치
- 저장된 장소 삭제

추후 구현:

- 저장 기록 수정 폼
- 하트 마커 이미지 에셋 고도화

## 비기능 요구사항

- 장소 저장은 네트워크가 느려도 입력 내용을 잃지 않아야 한다.
- 위치 권한을 거부해도 앱을 사용할 수 있어야 한다.
- 사진 접근 권한을 거부해도 메모와 장소만 저장할 수 있어야 한다.
- 지도 API 키는 앱 코드에 직접 노출하지 않는 방식을 검토한다.
- API 키, 토큰, 개인 설정 파일은 저장소에 커밋하지 않는다.
- SQLite에는 데이트 기록에 필요한 최소 개인정보만 저장한다.
- 현재 위치는 사용자가 명시적으로 저장하지 않는 한 데이트 카드 데이터로 저장하지 않는다.
- 추후 로그인과 커플 연동을 고려해 소유자와 공유 권한 필드를 확장 가능하게 둔다.
